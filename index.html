Go-Back-N Protocol Concept

What is Go-Back-N Protocol?

Go-Back-N is a sliding window protocol that belongs to Automatic Repeat reQuest (ARQ) protocols used in data link and transport layers for reliable data transmission.

In Go-Back-N, the sender can transmit multiple frames before receiving an acknowledgment, but is constrained to have no more than a maximum number of frames (the window size) outstanding.

Key Characteristics

  • Window Size: Defines the maximum number of frames that can be sent without receiving an acknowledgment
  • Sequence Numbers: Frames are numbered to maintain order and track acknowledgments
  • Cumulative Acknowledgment: A single ACK confirms all frames up to a specific sequence number
  • Timeout Mechanism: If an acknowledgment is not received within a specified timeout period, all unacknowledged frames are retransmitted

How Go-Back-N Works

Sender's Perspective:

  1. Maintains a window of up to N unacknowledged frames
  2. Continues sending frames until the window is full
  3. When an acknowledgment is received, the window slides forward
  4. If a timeout occurs before receiving an ACK, the sender retransmits all frames in the current window

Receiver's Perspective:

  1. Accepts only frames arriving in exact sequence order
  2. Discards out-of-order frames without buffering them
  3. Sends acknowledgment for the highest in-sequence frame received

Advantages and Disadvantages

Advantages:

  • Simple implementation, especially at the receiver
  • Efficient for low error rates
  • Better channel utilization compared to Stop-and-Wait

Disadvantages:

  • Wasteful when errors occur, as correctly received frames may be retransmitted
  • Not efficient in high error rate environments
  • Can lead to unnecessary retransmissions when only one frame is corrupted
